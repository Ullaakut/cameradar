# Security Policy

## Supported Versions

Cameradar is an open source project maintained by volunteers. Security fixes are generally only provided for the latest released version.

| Version        | Supported          |
| -------------- | ------------------ |
| Latest release | :white_check_mark: |
| Older releases | :x:                |

Users are encouraged to upgrade to the latest version as soon as possible when security updates are released.

## Reporting a Vulnerability

If you believe you have discovered a security vulnerability in Cameradar, please use GitHub's **Private Vulnerability Reporting** feature (Security tab → "Report a vulnerability") to submit your report.

Please include as much information as possible, including:

* A description of the vulnerability
* Steps to reproduce the issue
* A proof of concept, if available
* The affected version(s)
* Any relevant logs, screenshots, or network captures

### What to Expect

* We will acknowledge receipt of your report as soon as reasonably possible, typically within **14 days**.
* We will investigate the report and determine whether it constitutes a security vulnerability.
* If the report is accepted, we will work on a fix and coordinate disclosure with the reporter whenever practical.
* If the report is declined, we will provide an explanation whenever possible.

Please do **not** publicly disclose security vulnerabilities until a fix has been released or coordinated disclosure has been agreed upon.

We appreciate responsible disclosure and the efforts of security researchers who help keep Cameradar and its users secure.
